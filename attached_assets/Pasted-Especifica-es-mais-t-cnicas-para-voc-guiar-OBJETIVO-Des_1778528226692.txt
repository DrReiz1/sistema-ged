Especificações mais técnicas para você guiar: OBJETIVO

Desenvolver exclusivamente o protótipo visual front-end do sistema GED baseado no SDD previamente enviado, nos layouts do Figma e na referência visual/funcional do Paperless NGX.

Estamos atualmente apenas na fase de modelagem visual do sistema.

Ignore backend, autenticação real, banco de dados e integrações.

O foco é exclusivamente UX/UI profissional e navegação visual funcional.

STACK OBRIGATÓRIO

* Next.js 15
* React
* TypeScript
* Tailwind CSS
* shadcn/ui
* Lucide React Icons
* Framer Motion apenas para micro animações suaves
* App Router
* Estrutura modular por componentes

Não utilize Bootstrap, Material UI, CSS puro extenso ou bibliotecas antigas.

ARQUITETURA FRONT-END

Estruture o projeto como um sistema SaaS enterprise moderno:

* Sidebar fixa à esquerda
* Topbar superior
* Navegação SPA simulada
* Componentização reutilizável
* Layout responsivo desktop-first
* Estrutura organizada em:

  * components/
  * app/
  * modules/
  * layouts/
  * mock/
  * hooks/

Utilize dados mockados apenas para representação visual.

Não implementar backend.

DIRETRIZES VISUAIS

A interface deve seguir:

* estilo enterprise moderno
* visual semelhante ao Paperless NGX
* identidade visual baseada nas cores institucionais da TSEA
* baixa poluição visual
* foco em produtividade operacional
* aparência corporativa industrial moderna
* design clean e sofisticado
* espaçamentos consistentes
* tipografia profissional
* cards suaves
* sombras discretas
* bordas arredondadas moderadas
* tabelas modernas
* modais elegantes
* componentes altamente reutilizáveis

Utilize o Figma enviado como principal referência visual de:

* posicionamento
* cores
* hierarquia
* layout
* espaçamento

TELAS OBRIGATÓRIAS

* logo oficial da TSEA
* campo e-mail corporativo
* campo senha
* botão “Entrar”
* visual semelhante ao Figma enviado

Após login, redirecionar automaticamente para “Início”.

SIDEBAR

* ícones modernos

* estados ativos

* sessões organizadas

Menus:

* Início

Use o Paperless NGX como principal referência funcional.

Todo o esforço deve estar concentrado em:

* UX

* consistência visual
* experiência enterprise profissional
